### 1. Python 스코프 버그 수정 — `pre_analyzed_floor` 파싱 실패

**파일:** `backend/routers/pipeline.py`

**문제**
배치불가 구역을 설정하고 "배치 시작하기"를 누르면 아래 오류 발생:
```
분석 실패: pre_analyzed_floor 파싱 실패: cannot access local variable
'FloorAnalysis' where it is not associated with a value
```

**원인**
Python의 LEGB 스코프 규칙. 함수 내부 `else` 블록에서 `FloorAnalysis`를 로컬 임포트하면,
Python은 해당 이름을 함수 전체에서 "로컬 변수"로 간주한다.
그래서 `else` 블록 **위쪽**에서 `FloorAnalysis`를 참조하는 코드가
"아직 할당되지 않은 로컬 변수"를 참조하는 것으로 처리되어 에러 발생.

```python
# Before — 문제 코드
else:
    from agents.agent2_floor import FloorAnalysis, ReferencePoint, ConfidenceLevel
    #                               ^^^^^^^^^^^^^ 이 로컬 임포트가 함수 전체를 오염

# After — 수정
else:
    from agents.agent2_floor import ReferencePoint, ConfidenceLevel
    # FloorAnalysis는 파일 상단에서 이미 임포트되어 있으므로 제외
```

**기술 포인트:** Python LEGB(Local-Enclosing-Global-Builtin) 스코프, 조건부 임포트 패턴

### 2. 배치 금지 구역 — 벽 인접 배치 시 zone 제약 누락 수정

**파일:** `backend/core/spatial.py`

**문제**
처음 배치에서는 배치불가 구역을 잘 인식하고 비워두지만,
"입구 왼쪽벽에 상품진열대 추가해달라"는 요구사항을 적용하면 dead zone을 무시하고 배치.

**원인**
`try_place_cluster()` 내부에 두 개의 검증 함수가 존재:
- `_is_valid()` — 그리드 스캔 경로. `zone_constraints` 체크 **포함** ✅
- `_wall_valid()` — 벽 인접(wall snap) 경로. `zone_constraints` 체크 **누락** ❌

"입구 왼쪽벽에 붙여달라"처럼 위치가 명시된 요청은 wall snap 전략을 우선 실행하기 때문에
zone 체크를 건너뛰고 dead zone에 배치해버렸다.

```python
# Before — _wall_valid 내부
def _wall_valid(ccx, ccy):
    for poly, _, _ in built:
        if not room_poly.contains(poly): return False
        if any(poly.intersects(dz) for dz in dead_zones): return False
        if any(poly.intersects(pp) for pp in placed_polys): return False
        # zone_constraints 체크 없음!

# After
def _wall_valid(ccx, ccy):
    for poly, _, _ in built:
        if not room_poly.contains(poly): return False
        if any(poly.intersects(dz) for dz in dead_zones): return False
        if any(poly.intersects(pp) for pp in placed_polys): return False
        if zone_constraints and not _zone_allows(poly, "product_display", zone_constraints):
            return False  # ← 추가
```

**기술 포인트:** Shapely Polygon 교차 검사, 클로저(내부 함수)의 스코프 캡처

### 3. 클러스터 배치 실패 fallback — 크기 축소 재시도

**파일:** `backend/core/spatial.py`

**문제**
"오른쪽 벽에 상품진열대 최대한 채워달라"처럼 많은 수량을 요청하면,
`_compute_wall_capacity()`가 계산한 개수(예: 7개)를 하나의 클러스터로 묶어 배치 시도.
클러스터 폭(7 × 950mm = 6650mm)이 남은 공간에 맞지 않으면 **전체 실패**로 처리되었다.

**수정 내용**
클러스터가 실패하면 크기를 절반씩 줄여 재시도하는 fallback 루프 추가.
성공한 수만큼만 배치하고, 나머지는 "공간 부족으로 일부만 배치"로 실패 처리.

```python
# Before — 실패 시 바로 포기
if result_units is None:
    for intent in sub_group:
        failed_objects.append({...})
    continue

# After — 절반씩 줄여 재시도
if result_units is None:
    reduced = list(sub_group)
    while len(reduced) > 1 and result_units is None:
        reduced = reduced[:len(reduced) // 2]
        reduced_units = plan_cluster_layout(len(reduced), pd_w, pd_d)[0]
        result_units, cx, cy = try_place_cluster(
            ..., reduced_units, ..., zone_constraints=zone_constraints
        )
    if result_units is not None:
        # 초과분 실패 처리 후 배치 진행
        n = len(result_units)
        for intent in sub_group[n:]:
            failed_objects.append({"reason": "클러스터 크기 초과 — 공간 부족으로 일부만 배치", ...})
        sub_group = sub_group[:n]
    else:
        # 완전 실패
        ...
```

**기술 포인트:** 이진 감소(binary reduction) 재시도 패턴, product_display 클러스터 배치 알고리즘

### 4. 2D 결과 뷰에 배치 구역 시각화

**파일:** `frontend/src/App.tsx`

**문제**
배치 결과 화면에서 2D 뷰로 전환하면 오브젝트만 보이고,
사용자가 설정한 배치불가/포토존 구역이 보이지 않았다.

**수정 내용**
결과 `<FloorView2D>` 컴포넌트에 `zones`와 `deadZones` props 추가.
`userZones` state와 `agent2Preview.dead_zones_mm`은 배치 시작 후에도 유지되므로 그대로 전달.

```tsx
// Before
<FloorView2D
  roomPolygon={result.room_polygon_mm || []}
  placedObjects={localPlaced}
  editMode={editMode}
  ...
/>

// After
<FloorView2D
  roomPolygon={result.room_polygon_mm || []}
  placedObjects={localPlaced}
  editMode={editMode}
  zones={userZones}                            // ← 사용자 정의 구역
  deadZones={agent2Preview?.dead_zones_mm ?? []}  // ← 구조적 데드존
  ...
/>
```

**시각 표현:**
| 구역 유형 | 색상 |
|---|---|
| 배치불가(dead zone) | 빨간색 반투명 + 빨간 테두리 |
| 포토존 전용 | 핑크색 반투명 + 핑크 테두리 |
| 제한없음(auto zone) | entrance/mid/deep 각각 초록/보라/주황 |

### 5. 미배치 오브젝트 드래그앤드롭 수동 배치

**파일:** `frontend/src/components/FloorView2D.tsx`, `frontend/src/App.tsx`

**배경**
AI 배치에 실패한 오브젝트를 사용자가 직접 2D 도면에 원하는 위치에 배치할 수 있어야 한다.

**구현**

| 레이어 | 변경 내용 |
|---|---|
| `FloorView2D.tsx` | `draggingObjectType`, `onObjectDrop` props 추가. SVG에 `onDragOver` / `onDragLeave` / `onDrop` 핸들러 연결. 드래그 중 마우스 위치에 오브젝트 크기 프리뷰(반투명 사각형) 렌더링 |
| `App.tsx` | `draggingFailedType` state. 미배치 목록 항목에 `draggable` 속성 + `onDragStart`/`onDragEnd`. `onObjectDrop` 콜백에서 `localPlaced` 추가, `result.failed`에서 해당 항목 제거 |

**데이터 흐름:**
```
미배치 목록 아이템
  └─ onDragStart → dataTransfer.set('application/x-object-type', type)
                 → setDraggingFailedType(type)  [ghost 프리뷰 활성화]

SVG 위에서 DragOver
  └─ e.preventDefault()  [drop 허용]
  └─ getSVGCoords(e) → setDropPreview([svgX, svgY])  [프리뷰 이동]

SVG Drop
  └─ dataTransfer.get('application/x-object-type')
  └─ toMM(svgX, svgY) → (mm_x, mm_y)  [SVG좌표 → mm 변환]
  └─ setLocalPlaced([...prev, newObject])  [배치]
  └─ setResult(prev → failed 제거)        [목록 갱신]
```

**좌표 변환:**
```
mm → SVG:  svgX = (mm_x - minX) * scale + PADDING
SVG → mm:  mm_x = (svgX - PADDING) / scale + minX
```

**UX 디테일:**
- 2D 뷰에서만 드래그 가능 (`draggable={viewMode === '2d'}`)
- 드래그 시작 시 커서 변경(grab → copy)
- 드롭 후 Ctrl+Z로 되돌리기 가능 (`pushHistory` 적용)
- 드롭 성공 시 헤더 카운트(성공/실패) 실시간 업데이트
