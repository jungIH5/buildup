# 2026-04-07 작업 로그

## 주요 작업: Intent Parser 설계 및 구현

---

## 배경 — 기존 문제

Agent 3가 사용자 자연어를 직접 받아 LLM이 `reference_point`를 자유롭게 선택하는 구조였음.
"입구 오른쪽"처럼 방향이 명시된 요구사항도 LLM이 잘못 해석하면 엉뚱한 벽에 배치되는 문제가 반복됨.

이전까지의 수정들(벽 정렬 순서, 버퍼 수식 등)은 **배치 실행 로직**을 고친 것이었고,
**의도 해석 로직**은 건드리지 않은 상태였음.

---

## 해결 방향 — Intent Parser 도입

자연어 해석을 LLM에만 맡기지 않고, 구조화된 파싱 단계를 중간에 삽입.

```
기존: User Input → Agent1 → Agent2 → Agent3 (자연어 그대로)
변경: User Input → Agent1 → Agent2 → Intent Parser → Agent3 (구조화된 인텐트)
```

---

## 배치 요구사항 패턴 분류

| position_case | 예시 | 처리 방법 |
|---|---|---|
| `entrance_relative` | 입구 오른쪽/왼쪽/맞은편 | 입구 픽셀 위치로 동서남북 변환 |
| `absolute_wall` | 왼쪽 벽, 북쪽 벽 | 픽셀 좌표계 직접 매핑 |
| `zone_based` | 깊은 곳, 입구 가까이 | zone_label 직접 지정 |
| `adjacent` | A 옆에 B | 인접 오브젝트 기준 |
| `unspecified` | 위치 언급 없음 | Agent 3 자유 배치 |

## 구현 내용

### 1. `backend/core/intent_parser.py` 신규 생성

- Haiku LLM으로 자연어 파싱 → `object_type`, `quantity`, `position_case`, `relative_direction` 추출
- `_RELATIVE_TO_WALL` 테이블로 입구 기준 상대 방향 → 실제 벽 방향 변환
  - 입구가 south 벽에 있을 때: right→east, left→west, facing→north
- `_find_ref_by_wall`로 벽 방향 → `reference_point` name 매핑
- fill(-1) 수량은 Intent Parser에서 벽 용량 계산으로 실제 숫자로 변환
- 파싱 실패 시 기존 자연어 직접 파싱으로 폴백 (안전성 유지)

### 2. `backend/agents/agent3_layout.py` 수정

- `run_agent3`에 `resolved_intents: list[ResolvedIntent]` 파라미터 추가
- `forced_refs` dict로 Intent Parser가 지정한 기준점 등록
- LLM 호출 후 강제 기준점 덮어쓰기:
  ```python
  if placement.object_type in forced_refs:
      placement.reference_point = forced_refs[placement.object_type]
  ```
- `resolved_intents` 없으면 기존 자연어 파싱 폴백

### 3. `backend/routers/pipeline.py` 수정

- Agent 2 실행 후, Agent 3 호출 전 `parse_intents()` 실행
- `/run`, `/layout_only` 두 엔드포인트 모두 적용
- Intent Parser 실패 시 경고 로그 후 폴백 (서비스 중단 없음)

## 발견된 버그 (미수정)

### 같은 오브젝트 타입에 두 위치 지정 시 덮어쓰기 문제

요구사항: "입구 왼쪽 벽에 최대한, 중앙에도 4개"

```python
forced_refs["product_display"] = "west_wall_mid"  # 1번 intent
forced_refs["product_display"] = "center"          # 2번 intent → 덮어씀
# 결과: 전체가 center 기준으로 배치됨
```

현상: 진열대가 벽이 아닌 공간 중앙에 가로로 일렬 배치됨

원인:
1. `forced_refs`가 `dict[object_type → ref]` 구조라 같은 타입은 마지막 값만 유지
2. `product_display`에 `center` 기준점을 주면 `_try_wall_snap`이 오작동

개선 방향:
- `forced_refs`를 dict에서 **배치 그룹 리스트**로 변경
- 같은 오브젝트 타입이라도 위치가 다르면 별도 그룹으로 분리 처리
- Agent 3에 그룹별 독립 PlacementIntent 생성해서 전달
